    • Deployment Steps

    Now that it is in the ACR you need to redirect the image in the YAML file towards the ACR

    After finding the twistlock/defender Image the information after the docker pull is what you will need to replace the Image line with on the YAML file.

    Save the YAML file on the home directory on you Linux machine

    After you replace the twistlock image directory with the ACR image path you will pull and deploy it

    Run these commands:

    - $ kubectl create namespace twistlock

    - $ kubectl create -f %file path% -n twistlock

    - $ kubectl get -n twistlock po

    - $ kubectl logs %podname% -n twistlock

    %filepath% is where you have your YAML file (./daemonset.yaml)

    %podname% is the name of one of your pods

    If there are no logs upon first creation, then you have deployed it correctly and should find it on Prisma Compute
